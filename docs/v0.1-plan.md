# v0.1 Plan

## Problem

Kevin wants AI to think proactively, find useful work, and avoid waiting for
explicit commands, while still preserving safety and Kevin's decision style.

## User Goal

An always-available assistant that notices useful work, ranks it, and prepares
safe next actions using Kevin's persona.

## MVP Scope

1. Read Kevin persona.
2. Read configured repo rules.
3. Scan repositories in read-only mode.
4. Produce candidate tasks.
5. Classify tasks by autonomy and risk.
6. Generate OpenCode prompts.
7. Produce a concise Traditional Chinese report.

## Candidate Task Sources

1. TODO/FIXME markers.
2. Dirty git worktrees.
3. Recent failed checks if logs are available.
4. Docs that mention old behavior or stale paths.
5. Missing README/setup instructions.
6. Repeated manual workflows described in docs or issues.

## Task Scoring

```text
score =
  user_experience_value
+ stability_value
+ verifiability_value
+ automation_value
+ prototype_value
- regression_risk
- approval_required_weight
- cost_or_deployment_risk
```

## Report Format

```text
今日建議
- <task>: <why>, <risk>, <next action>

可自動候選
- ...

需要 Kevin 決策
- ...

暫不做
- ...

證據
- scanned repos, commands, files, persona version
```
